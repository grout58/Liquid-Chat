## What's New — v1.1 (March 2026)

### Reliability
- **Auto-Reconnect** — Exponential backoff reconnection (5s → 10s → … → 5min cap). Reconnects silently; alerts you only when the connection persistently fails.
- **Channel Auto-Rejoin** — Previously joined channels are automatically rejoined after reconnect. No manual re-joining needed.
- **Connection Error Alerts** — Persistent failures surface as dismissible alerts with Retry / Stop Retrying actions.

### New Features
- **Smart Reply Suggestions** — AI-powered quick reply chips appear below the message list, generated by Apple Intelligence based on recent conversation context.
- **Pastebin Integration** — Pasting 5+ lines prompts to upload to [paste.rs](https://paste.rs) and send the link instead, keeping channels clean.
- **IRCv3 Batch Visual Grouping** — Replayed history (e.g. CHATHISTORY batch) is clearly separated with a "replayed history" divider.
- **URL Preview Thumbnails** — Images and links show inline previews with title, description, and thumbnail.
- **Tab Completion** — Press Tab to complete nicknames; cycles through matches, appending `:` at line start.
- **Nicklist Context Menu** — Right-click any user for Whois, Query (DM), Ignore, Op/Deop, Kick/Ban.
- **WHOIS Display** — Whois responses appear as system messages directly in the relevant channel.
- **Configurable Message History** — Settings control how many messages are retained per channel.
- **Per-Channel Logging** — All messages logged to `~/Library/Application Support/Liquid Chat/Logs/{server}/{channel}/`.

### Bug Fixes (9 confirmed)
- **Data race** — `IRCConnection.state` was written from Network.framework's background queue; all mutations now dispatched to main thread
- **SASL chunking** — Base64 payloads of exactly 400 bytes didn't send the required trailing `+`
- **Timer leak** — CAP negotiation timeout wasn't cancelled on `disconnect()`
- **Integer overflow** — `abs(Int.min)` crash in NicknameColorizer replaced with `.magnitude`
- **Memory leak** — URL preview cache was unbounded; now capped at 200 entries with LRU eviction
- **Regex recompilation** — `NSRegularExpression` was compiled fresh on every URL parse; now cached
- **Missing timeout** — URL preview fetches had no timeout; switched to 10s/15s custom URLSession
- **Concurrency** — `ChannelLogger.deinit` accessed actor-isolated state from a nonisolated context
- **Dead code** — `cachedRegexPatterns` static in URLPreviewFetcher was never used; removed

## IRC Protocol Implementation

### Connection Handshake

```
1. CAP LS 302                    // Request IRCv3 capabilities
2. PASS password (if needed)     // Server password / bouncer auth
3. NICK nickname                 // Set nickname
4. USER username 0 * :realname   // User identification
5. CAP REQ :sasl server-time … // Request capabilities
6. CAP END                       // End capability negotiation
```

### Message Format (RFC 1459 + IRCv3)

```
[@tag=value;tag2=value2] [:prefix] COMMAND [params] [:trailing]
```

**Examples:**
```
@time=2026-03-01T12:00:00.000Z :nick!user@host PRIVMSG #channel :Hello!
:server.com 001 mynick :Welcome to the IRC Network
PING :server.com
```

### Supported Commands

**User Commands:**
- `/join #channel` — Join a channel
- `/part [reason]` — Leave current channel
- `/msg nick message` — Send private message
- `/me action` — Send action message
- `/whois nick` — Get user information (shown inline)
- `/list` — Browse available channels (handles 10,000+)
- `/topic [new topic]` — View or set channel topic
- `/kick nick [reason]` — Kick user from channel
- `/ban hostmask` — Ban user
- `/quit [message]` — Disconnect from server

**Automatic Handling:**
- PING/PONG — Keep-alive
- CAP — IRCv3 capability negotiation
- BATCH — Grouped message delivery with visual separators
- server-time — Accurate message timestamps
- Auto-reconnect — Exponential backoff on disconnect
- Auto-rejoin — Restores channel list after reconnect

## AI Features

### Smart Reply Suggestions

Quick reply chips appear automatically after a message arrives from someone else. Powered by Apple Intelligence on macOS 26+; falls back to rule-based suggestions on earlier systems.

- Tap a chip to populate the input field instantly
- Regenerates after each new message (1.5s debounce)
- Respects the AI temperature setting in Settings > Advanced

### Smart Channel Recommendations

Analyzes your recent conversation and suggests related channels with relevance scores. One-click to join.

### Chat Summarization

Generates structured summaries of missed conversations:
- Key discussion points (3–5 bullets)
- Main topics and sentiment
- Action items and TODOs

### Configuration

Settings > Advanced:
- **Enable AI Features** — Master toggle
- **Enable Smart Replies** — Toggle quick reply suggestions
- **AI Temperature** — Control creativity (0.0–1.0)
- **Auto-Summarize Threshold** — Message count trigger

## Building and Running

### Requirements

- **macOS 26.0 or later** (Liquid Glass + Apple Intelligence)
- **Xcode 26.3 or later**
- **Swift 6.0 or later**
- Apple Intelligence enabled (for AI features; app works without it)

### Build Steps

```bash
git clone https://github.com/grout58/Liquid-Chat.git
cd Liquid-Chat
open "Liquid Chat.xcodeproj"
# Build and run: ⌘R
```

### Quick Start

1. Launch Liquid Chat
2. Click **New Connection…** (⌘N)
3. Enter server details — e.g. `irc.libera.chat`, port `6697`, SSL on
4. Click **Connect**
5. Join a channel: `/join #swift`
6. Try AI features:
   - **Search**: ⌘F
   - **Summarize**: sparkles button in toolbar
   - **Smart Replies**: appear automatically below messages
   - **Recommendations**: star button in toolbar
